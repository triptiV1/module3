# CSC580 CTA 3.2 – Predicting Fuel Efficiency Using TensorFlow

This assignment builds a TensorFlow regression model (tf.keras) on the UCI Auto MPG dataset to predict fuel efficiency (MPG) for late-1970s and early-1980s automobiles.

## Contents
- `auto_mpg_regression.py` – Main script that runs the full pipeline and produces all outputs.
- `requirements.txt` – Python dependencies for a clean run.
- `outputs/` – All generated artifacts (pairplot, model summaries, training plots, history tails, Word report).
- `CSC580_CTA_3_2_Vishwakarma_Tripti.zip` – Submission archive created by the script (can be regenerated).

## Environment Setup
Recommended to use a virtual environment:

```bash
python3 -m venv .venv_cta32
source .venv_cta32/bin/activate
pip install -r requirements.txt
```

## How to Run
Default training runs for 1000 epochs and generates all artifacts and the final ZIP.

```bash
# From this directory
python auto_mpg_regression.py --epochs 1000
```

Optional flags:
- `--epochs N` – Set custom number of epochs (e.g., 300 for quick test).
- `--no-docx` – Skip creating the Word report.

## Generated Outputs (outputs/)
- `pairplot.png` – Seaborn pairplot of selected features.
- `dataset_tail.txt` – Tail of the cleaned dataset after one-hot encoding.
- `train_stats_tail.txt` – Tail of the train statistics (with MPG removed) used for normalization.
- `model_summary_mse.txt` – Model summary for MSE-loss model.
- `model_summary_mae.txt` – Model summary for MAE-loss model.
- `history_mae.png` – MAE training/validation curves (both models overlaid).
- `history_mse.png` – MSE training/validation curves (both models overlaid).
- `history_tail_mse.txt` – Tail of training history for MSE-loss model.
- `history_tail_mae.txt` – Tail of training history for MAE-loss model.
- `CSC580_CTA_3_2_Vishwakarma_Tripti.docx` – Word report auto-generated by the script.

## Notes
- The script uses z-score normalization (mean/std) from the training split.
- Two identical architectures are trained with different losses:
  - MSE-loss model (penalizes larger errors more strongly).
  - MAE-loss model (more robust to outliers).
- The script prints test-set metrics for both models at the end.
